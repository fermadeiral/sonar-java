Calling "org.junit.Assert.fail()" (JUnit 4) or "org.junit.jupiter.api.Assertions.fail()" (JUnit 5) throws a "java.lang.AssertionError". If this call is done within the try block of a try-catch cathing a similar error, you should make sure to test some properties of the exception. Otherwise, the test will never fail.

Noncompliant Code Example

@Test
public void should_throw_assertion_error() {
  try {
    throwAssertionError(); // This test will pass even if we comment this line!
    Assert.fail("Expected an AssertionError!");
  } catch (AssertionError e) {}
}

private void throwAssertionError() {
  throw new AssertionError("My assertion error");
}

Compliant Solution

assertThrows(AssertionError.class, () -> throwAssertionError());

try {
   throwAssertionError();
   Assert.fail("Expected an AssertionError!");
 } catch (AssertionError e) {
   Assert.assertThat(e.getMessage(), is("My assertion error"));
 }
